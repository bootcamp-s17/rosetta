Classes

Javascript

Classes are "special functions". Just as you can define function expressions and function declarations, the class syntax has two components: class expressions and class declarations.

Declarations

Not "Hoisted", meaning order matters. Must be declared above where it is used. Normal functions are hoisted in that you can call them regardless of where in the code they are declared.

Example:

            	
class Person {
	constructor(name, height, weight) {
	    this.name = name;
	    this.height = height;
	    this.weight = weight;
	  }
}

Contains a constructor that builds up the class members. Can also create functions that act on class members.

			
	get bmi() {
		return this.calcBMI();
	}

	calcBMI() {
		return this.weight/(this.height*this.height);
	}

You can create new members of the class,

var godzilla = new Person('Godzilla', 50, 148778297.36)

and access the function with

console.log(godzilla.bmi);
Extends

You can also carry over properties from another class. If you have a class Kaiju with property isMonster = True;. Then when you declare a new one, you use class godzilla extends kaiju

PHP

Similar to JS, the PHP classes structure can be viewed as a "special function". The defined class holds definitions of the properties and methods belonging to that class. 

            

class SimpleClass{
	    // property declaration
	    public $var = 'a default value';

	    // method declaration
	    public function displayVar() {
	        echo $this->var;
	    }
	}

In the class below, variables are declared as private, protected, and publuc. These dictate where these attributes can be altered, or functions called:

  • Public: Can be called anywhere, outside of the class, inside the class, or even in a child.
  • Protected: Can be called from only within the current class or any children of that class (eg: gender can be altered by Person or Man)
  • Private: Can only be called by the containing class. (eg: alive can only be altered by Person)

Calling a constructor in PHP is slightly different as well. __construct() defines the class properties that will be manipulated. In addition, just as JS extends to another class to allow these properties to be used elsewhere, PHP uses the same method to allow the class Man, to use the properties in the class Person, even the private ones. 

		

class Person {
    private $alive = true;

    protected $gender = '';
    protected $genitalOrgan = ''; 
}

class Man extends Person {
    public function __construct(){
        $this->gender = 'male';
        $this->genitalOrgan = 'penis';
    }
}
